Dokumentasi Proyek Grafkom Start Menu

C++ WinBGIm graphics.h

Proyek ini adalah simulasi antarmuka Windows XP Start Menu dan Desktop yang dibuat untuk mata kuliah Grafika Komputer. Program menggunakan library WinBGIm (graphics.h) untuk menggambar grafik 2D secara langsung tanpa akselerasi GPU. Dokumentasi ini menjelaskan setiap file secara mendalam — bukan hanya apa yang dilakukan kode, tetapi juga mengapa implementasi tersebut dipilih.

main.cpp — Entry Point & Game Loop

1. Tujuan File

main.cpp adalah pusat kendali seluruh program. File ini menginisialisasi jendela grafis 1280x720, menggambar tampilan awal, menyimpan buffer desktop, dan menjalankan game loop utama yang menangani input serta me-redraw layar ~60 kali per detik menggunakan double buffering.

💡 Konsep Kunci: Game Loop

Dalam pemrograman grafis (dan game), "game loop" adalah pola di mana program terus berulang: membaca input → memperbarui state → menggambar ulang layar → menunggu sebentar. Ini memastikan tampilan selalu responsif terhadap perubahan. Bayangkan seperti flip book — setiap halaman adalah satu frame, dan ketika dibalik cepat, gambarnya terlihat bergerak.

Mengapa static di level file? Kata kunci static pada variabel global membatasi linkage-nya hanya ke file ini (internal linkage). Variabel ini tidak bisa diakses dari file .cpp lain. Ini lebih aman daripada variabel global biasa karena mencegah modifikasi tak sengaja dari modul lain.

4. Fungsi main() — Penjelasan Mendalam

Tujuan

Entry point program. Menginisialisasi, menjalankan game loop, dan membersihkan resources saat selesai.

Parameter

Tidak ada (standard main() tanpa command-line arguments).

Nilai Kembali

0 = program selesai normal.

Alur Kerja Langkah Demi Langkah

  1. Inisialisasi Window — initwindow(1280, 720, "Start Menu Clone") membuat jendela grafis. Mengapa 1280x720? Resolusi HD standar yang cukup untuk menampilkan desktop + taskbar + start menu tanpa scroll.
  2. Gambar Desktop — Memanggil drawDesktop() untuk menggambar wallpaper dan ikon.
  3. Simpan Desktop ke Buffer — imagesize() menghitung ukuran memori yang dibutuhkan, malloc() mengalokasikannya, getimage() menyalin piksel dari layar ke buffer. Ini adalah optimasi kunci: desktop hanya perlu digambar sekali (melibatkan pembacaan file .bmp yang lambat), lalu tinggal di-copy dari buffer setiap kali redraw diperlukan.
  4. Double Buffer Setup Awal — Menggambar di halaman tersembunyi, lalu menukar ke halaman yang terlihat.
  5. Game Loop — Loop while (isRunning) berjalan terus:
    • Membaca input mouse (handleMouse()) dan keyboard (handleKeyboard())
    • Mengecek perubahan waktu (isTimeChanged()) untuk update jam
    • Mengecek status animasi Start Menu (getAnimProgress())
    • Mendeteksi perubahan hover state (isHoveringProgramsChanged())
    • Menentukan flag redraw berdasarkan semua perubahan di atas
    • Jika redraw diperlukan: restore desktop dari buffer → gambar taskbar → gambar start menu → tukar halaman buffer
    • delay(16) — menunggu 16 milidetik (~60 FPS)
  6. Cleanup — free(bgImage) melepaskan memori, closegraph() menutup jendela.
ℹ Mengapa 60 FPS?

Layar monitor umumnya me-refresh 60 kali per detik (60 Hz). Dengan menggambar 60 frame per detik, animasi terlihat mulus. delay(16) (16 ms ≈ 1000/60) adalah pendekatan sederhana. Namun, ini tidak memperhitungkan waktu yang dihabiskan untuk operasi lain (menggambar, input handling), sehingga FPS aktual bisa lebih rendah.

5. Blok Kode Penting

5.1 Double Buffering

cpp
// Halaman aktif untuk menggambar (tidak terlihat)
setactivepage(1 - page);

// Restore desktop dari buffer
putimage(0, 0, bgImage, COPY_PUT);

// Gambar elemen UI
drawTaskbar();
drawStartMenu();

// Tukar ke halaman yang terlihat
setvisualpage(1 - page);
page = 1 - page;  // Toggle halaman

Analogi: Bayangkan seorang pesulap yang memiliki dua papan tulis. Penonton hanya melihat satu papan. Pesulap menggambar di papan tersembunyi, lalu ketika selesai, langsung menukar papan. Penonton tidak pernah melihat proses menggambar, hanya hasil akhirnya. Inilah prinsip double buffering.

5.2 Deteksi Perubahan State (Dirty Flag Pattern)

cpp
bool mouseAction = handleMouse();
bool keyAction = handleKeyboard();
bool timeChanged = isTimeChanged();
float currentAnimProgress = getAnimProgress();

// Deteksi animasi yang sedang berjalan
bool isAnimating = (currentAnimProgress > 0.0f && currentAnimProgress < 1.0f)
                   || (currentAnimProgress != lastAnimProgress);

// Deteksi tepat satu frame saat animasi selesai
static bool wasAnimating = false;
bool animationJustFinished = (wasAnimating && !isAnimating);
wasAnimating = isAnimating;

needsMenuRedraw = mouseAction || keyAction || isAnimating || hoverChanged || animationJustFinished;
needsClockRedraw = timeChanged;
needsFullRedraw = needsMenuRedraw || needsClockRedraw;

Mengapa deteksi "animationJustFinished"? Tanpa ini, ketika animasi selesai (animProgress mencapai 0.0 atau 1.0), isAnimating menjadi false, dan tidak ada redraw. Ini menyebabkan menu "menghilang tiba-tiba" tanpa frame terakhir. Deteksi animationJustFinished memastikan frame terakhir animasi tetap digambar.

6. Konsep C++ & WinBGIm pada File Ini

  • Double Buffering — setactivepage() / setvisualpage(): WinBGIm mendukung hingga 4 halaman grafis. Halaman aktif = tempat menggambar (tidak terlihat), halaman visual = yang ditampilkan ke layar.
  • imagesize() / getimage() / putimage() — Menyimpan dan me-restore area layar ke/dari memori. imagesize() mengembalikan jumlah byte yang dibutuhkan. getimage() menyalin piksel ke buffer. putimage() dengan flag COPY_PUT menyalin buffer ke layar.
  • malloc() / free() — Alokasi memori manual gaya C. Di C++ modern, new/delete atau std::vector lebih disarankan.
  • static variabel lokal — Nilainya bertahan antar pemanggilan fungsi. Berguna untuk state tracking seperti wasAnimating.

7. Saran Perbaikan

  • Gunakan std::vector<unsigned char> alih-alih malloc() — Lebih idiomatis C++, RAII (otomatis dibersihkan), type-safe. Namun fallback saat malloc gagal sudah baik.
  • Pisahkan game loop — main() melakukan terlalu banyak. Fungsi run() atau appLoop() akan meningkatkan readability.
  • Delta time — delay(16) mengabaikan waktu eksekusi kode lain. Gunakan clock() untuk menghitung delta time aktual agar frame rate lebih konsisten.
  • Gunakan enum — Flags needsFullRedraw dll bisa digabung menjadi enum class RedrawState.

3. Fungsi drawDesktop()

Tujuan

Menggambar seluruh area desktop satu kali. Hasilnya disimpan ke buffer oleh main.cpp untuk digunakan sepanjang program.

Alur Kerja Langkah Demi Langkah

  1. Fallback background — Menggambar kotak biru (BLUE) memenuhi seluruh layar dengan bar(0, 0, getmaxx(), getmaxy()). Ini adalah jaring pengaman jika wallpaper gagal dimuat.
  2. Load wallpaper — readimagefile("assets/wallpaper.bmp", 0, 0, getmaxx(), getmaxy()) membaca file .bmp dan menggambarnya di atas background biru. Fungsi ini tidak memberi tahu jika file tidak ditemukan; ia hanya tidak menggambar apa-apa.
  3. Gambar 5 ikon — Setiap ikon: panggil readimagefile() untuk gambar 40x60px, lalu outtextxy() untuk teks label di bawahnya. Koordinat Y setiap ikon bergeser ~110px: 20, 130, 240, 350, 460.

4. Blok Kode Penting

Fallback Background

cpp
// Fallback background color (biru)
setfillstyle(SOLID_FILL, BLUE);
bar(0, 0, getmaxx(), getmaxy());

// Coba load wallpaper jika ada
readimagefile("assets/wallpaper.bmp",
                  0, 0,
                  getmaxx(), getmaxy());

Mengapa fallback dan bukan cek error? readimagefile() tidak mengembalikan kode error. Jika file tidak ada, fungsi ini diam-diam tidak menggambar apa pun. Background biru memastikan layar tidak kosong (hitam).

Pola Penggambaran Ikon

cpp
readimagefile("assets/browser.bmp",
                  20, 20,   // kiri-atas
                  80, 80);  // kanan-bawah
outtextxy(25, 90, (char*)"Browser");

Mengapa cast (char*)? Fungsi outtextxy() adalah warisan dari C lama yang mengharapkan char* non-const. String literal di C++ bertipe const char*, sehingga perlu cast. Di library modern, parameter ini seharusnya const char*.

5. Konsep WinBGIm

  • readimagefile() — Membaca file .bmp dan langsung menggambarnya ke layar dengan scaling otomatis. Mendukung format Windows BMP.
  • getmaxx() / getmaxy() — Mengembalikan lebar/tinggi maksimum area gambar dalam piksel. Berguna untuk menggambar elemen yang mengisi seluruh layar.
  • setfillstyle() + bar() — setfillstyle mengatur pola dan warna isian, bar() menggambar kotak terisi.
  • outtextxy(x, y, text) — Menggambar teks di koordinat (x,y) menggunakan font bitmap default 8x8.

6. Saran Perbaikan

  • Cek file dengan std::ifstream — Sebelum readimagefile(), cek apakah file ada. Jika tidak, tampilkan placeholder atau pesan.
  • Gunakan array ikon + loop — Lima ikon digambar dengan kode hampir identik. Struktur data:
    cpp
    struct Icon { const char* path; int x, y; const char* label; };
Icon icons[] = {
  {"assets/browser.bmp",  20, 20,  "Browser"},
  {"assets/folder.bmp",   20, 130, "Folder"},
  // ...dan seterusnya
};
for (auto& icon : icons) {
    readimagefile(icon.path, icon.x, icon.y, icon.x+60, icon.y+60);
    outtextxy(icon.x+5, icon.y+70, (char*)icon.label);
}
    Ini akan mengurangi duplikasi kode secara signifikan.

5. Fungsi drawStartMenu() — Jantung Start Menu

Tujuan

Menggambar seluruh Start Menu: header, dua kolom, footer, item menu, dan pop-up "All Programs". Dipanggil setiap frame (dalam game loop) karena animasi dan hover berubah terus-menerus.

Alur Kerja Langkah Demi Langkah

  1. Update Animasi — Jika menu terbuka (startMenuOpen == true), animProgress naik dari 0 ke 1. Jika tertutup, turun dari 1 ke 0. Kecepatan: animSpeed = 0.1f per frame (~10 frame untuk animasi penuh).
  2. Early Exit — Jika animProgress <= 0, return tanpa menggambar apa pun (menu sepenuhnya tersembunyi).
  3. Hitung Dimensi Menu — Tinggi menu = targetHeight * animProgress. Menu "tumbuh" dari bawah (taskbar) ke atas.
  4. Gambar Header — Kotak biru di atas menu. Jika animasi > 20%, tampilkan teks "Administrator" dan avatar placeholder (kotak abu-abu).
  5. Gambar Kolom Kiri (Putih) — bar() dari bawah header ke atas footer.
  6. Gambar Kolom Kanan (Biru Muda) — bar() di sebelah kanan kolom kiri.
  7. Gambar Footer — Kotak biru di bawah menu.
  8. Gambar Item Menu — Jika animasi > 80%, gambar semua item menggunakan drawMenuItem().
  9. Deteksi Hover "All Programs" — Cek apakah mouse di atas area tombol "All Programs".
  10. Update Animasi Pop-Up — Sama seperti menu utama, tapi lebih cepat (animSpeed * 1.5).
  11. Gambar Pop-Up — Jika popupAnimProgress > 0: shadow (kotak abu-abu offset), background putih, border biru, dan item menu pop-up.
  12. Update State Tracking — Simpan lastHoveringPrograms, lastMouseX, lastMouseY untuk deteksi perubahan di frame berikutnya.

6. Blok Kode Penting

6.1 Animasi Interpolasi Linear

cpp
float animSpeed = 0.1f;
if (startMenuOpen) {
    if (animProgress < 1.0f) animProgress += animSpeed;
    if (animProgress > 1.0f) animProgress = 1.0f;  // Clamp
} else {
    if (animProgress > 0.0f) animProgress -= animSpeed;
    if (animProgress < 0.0f) animProgress = 0.0f;  // Clamp
    isHoveringPrograms = false;  // Reset saat menu tertutup
}

Mengapa linear interpolation (LERP)? Ini adalah pendekatan paling sederhana: setiap frame, progress berubah sebesar konstanta. Hasilnya adalah gerakan dengan kecepatan konstan. Kekurangannya: terasa "kaku" dibanding easing functions (ease-in-out). Namun untuk proyek pembelajaran, LERP sudah sangat baik karena mudah dipahami dan di-debug.

Mengapa clamp? Tanpa clamp, progress bisa melebihi 1.0 atau kurang dari 0.0 karena floating-point. Clamp menjaga nilai tetap dalam rentang [0, 1].

6.2 Layout Menu Dinamis

cpp
int startY = getmaxy() - 50;                // Posisi Y taskbar
int targetHeight = 450;                       // Tinggi penuh menu
int currentHeight = (int)(targetHeight * animProgress); // Tinggi saat ini
int menuY = startY - currentHeight;             // Menu tumbuh ke atas

Mengapa startY - currentHeight? Menu tumbuh dari taskbar ke atas. Saat animProgress = 0, currentHeight = 0 dan menuY = startY (menu tersembunyi di balik taskbar). Saat animProgress = 1, currentHeight = 450 dan menuY = startY - 450 (menu terbuka penuh).

6.3 Hover Detection & Highlight

cpp
static bool drawMenuItem(int x, int y, int w, int h,
                         const char* text, int mx, int my,
                         int bgNormal, int fgNormal,
                         int bgHover, int fgHover,
                         int textX, int textY) {
    bool hovered = (mx >= x && mx <= x + w &&
                    my >= y && my <= y + h);
    if (hovered) {
        setfillstyle(SOLID_FILL, bgHover);
        bar(x, y, x + w, y + h);
        setbkcolor(bgHover);
        setcolor(fgHover);
    } else {
        setfillstyle(SOLID_FILL, bgNormal);
        bar(x, y, x + w, y + h);
        setbkcolor(bgNormal);
        setcolor(fgNormal);
    }
    outtextxy(textX, textY, (char*)text);
    return hovered;
}

Mengapa mengembalikan hovered? Nilai kembali memungkinkan pemanggil untuk melakukan aksi tambahan saat item di-hover. Contoh: saat "Turn Off Computer" di-hover DAN diklik, program keluar. Tanpa nilai kembali, pemanggil harus menghitung ulang bounding box.

6.4 Turn Off Computer — Klik untuk Keluar

cpp
if (drawMenuItem(menuX + 250, footerY + 5, 150, 30,
    "Turn Off Computer", mx, my,
    BLUE, WHITE, LIGHTBLUE, WHITE,
    menuX + 260, menuY + currentHeight - 30))
{
    if (ismouseclick(WM_LBUTTONDOWN)) {
        clearmouseclick(WM_LBUTTONDOWN);
        closegraph();
        exit(0);
    }
}

Mengapa clearmouseclick()? Event klik mouse bersifat queued — jika tidak di-clear, event yang sama bisa diproses lagi di frame berikutnya. clearmouseclick() menghapus event dari antrean setelah diproses.

6.5 Animasi Pop-Up dengan Shadow

cpp
// Shadow sederhana (offset 5px)
setfillstyle(SOLID_FILL, DARKGRAY);
bar(popupX + 5, popupY + popupTargetHeight - popupHeight + 5,
      popupX + popupWidth + 5, popupY + popupTargetHeight + 5);

// Background pop-up
setfillstyle(SOLID_FILL, WHITE);
bar(popupX, popupY + popupTargetHeight - popupHeight,
      popupX + popupWidth, popupY + popupTargetHeight);

// Border
setcolor(BLUE);
rectangle(popupX, popupY + popupTargetHeight - popupHeight,
            popupX + popupWidth, popupY + popupTargetHeight);

Teknik shadow: Shadow adalah kotak DARKGRAY yang digambar sebelum background putih, dengan offset 5px ke kanan-bawah. Background putih menimpa sebagian shadow, menciptakan efek bayangan. Ini adalah teknik shadow murah yang umum di GUI toolkit klasik.

6. Konsep C++ & WinBGIm

  • Linear Interpolation (LERP) — Animasi dengan kecepatan konstan. value += speed * deltaTime. Di sini, deltaTime implisit = 1 frame.
  • Clamping — Membatasi nilai dalam rentang [min, max] untuk mencegah overflow/underflow.
  • Bounding Box Hit Test — Mengecek apakah titik (mx, my) berada dalam kotak [x, x+w] x [y, y+h].
  • State Tracking — Menyimpan state sebelumnya (last*) untuk mendeteksi perubahan antar frame.
  • Shadow Technique — Menggambar kotak offset sebelum elemen utama untuk ilusi kedalaman.
  • Event Clearing — clearmouseclick() mencegah event yang sama diproses dua kali.

7. Saran Perbaikan

  • Easing functions — Ganti LERP dengan ease-out (perlambatan di akhir) agar animasi terasa lebih natural. Contoh: t = 1 - (1-t)^2 (quadratic ease-out).
  • Delta time — animSpeed = 0.1f per frame akan menghasilkan kecepatan berbeda di monitor 60Hz vs 144Hz. Gunakan delta time: animProgress += speed * deltaTime.
  • Pisahkan ke file sendiri — drawMenuItem() adalah helper generik yang bisa dipakai modul lain. Pertimbangkan memindahkannya ke utils.cpp.
  • Gunakan array untuk item menu — 15+ item menu digambar dengan kode hampir identik. Struktur data dan loop akan sangat mengurangi duplikasi.
  • Pisahkan pop-up ke fungsi terpisah — drawStartMenu() terlalu panjang (~200 baris). Fungsi drawPopupMenu() akan meningkatkan readability.
  • Hindari exit(0) di fungsi library — exit() langsung menghentikan program tanpa cleanup. Lebih baik set flag isRunning = false dan biarkan main() yang membersihkan.

clock.cpp — Jam Digital di Taskbar

1. Tujuan File

Menampilkan jam digital (HH:MM) dan tanggal (DD/MM/YYYY) di pojok kanan bawah taskbar. Juga menyediakan fungsi isTimeChanged() agar main loop tahu kapan harus me-redraw jam.

2. Penjelasan #include

IncludePenjelasan
#include <graphics.h>outtextxy(), textwidth(), getmaxx(), getmaxy().
#include <time.h>time(), localtime(), struct tm untuk mendapatkan waktu sistem.
#include <stdio.h>sprintf() untuk format string waktu & tanggal.
#include "../include/clock.h"Header sendiri.

3. Variabel Global Statis

cpp
static int lastDisplayedHour   = -1;  // Jam terakhir yang ditampilkan
static int lastDisplayedMinute = -1;  // Menit terakhir yang ditampilkan

Nilai awal -1 memastikan pendeteksian perubahan pada frame pertama (karena jam aktual tidak pernah -1).

4. Fungsi isTimeChanged()

Tujuan

Mengecek apakah waktu sudah berubah sejak pengecekan terakhir. Digunakan oleh main loop untuk menentukan apakah jam perlu di-redraw.

Nilai Kembali

true jika jam atau menit berbeda dari yang terakhir ditampilkan.

Alur Kerja

  1. Ambil waktu sistem saat ini dengan time(NULL) + localtime().
  2. Bandingkan tm_hour dan tm_min dengan lastDisplayedHour dan lastDisplayedMinute.
  3. Jika berbeda: update nilai terakhir, return true.
  4. Jika sama: return false (tidak perlu redraw).

5. Fungsi drawClock()

Tujuan

Menggambar waktu dan tanggal di pojok kanan bawah taskbar.

Alur Kerja

  1. Ambil waktu sistem.
  2. Format string dengan sprintf(): jam "HH:MM", tanggal "DD/MM/YYYY". Perhatikan tm_mon + 1 (bulan 0-based) dan tm_year + 1900 (tahun sejak 1900).
  3. Set warna teks putih, background DARKGRAY (samakan dengan taskbar).
  4. Hitung posisi: getmaxx() - 15 sebagai margin kanan.
  5. Gunakan textwidth() untuk right-align teks (tepi kanan teks = margin kanan layar).
  6. Jam di baris atas (getmaxy() - 38), tanggal di baris bawah (getmaxy() - 20).
  7. Restore background color lama setelah selesai (good practice).

6. Blok Kode Penting

cpp
char timeBuffer[10];
char dateBuffer[20];

sprintf(timeBuffer, "%02d:%02d", t->tm_hour, t->tm_min);
sprintf(dateBuffer, "%02d/%02d/%04d",
         t->tm_mday, t->tm_mon + 1, t->tm_year + 1900);

setcolor(WHITE);
int oldBk = getbkcolor();
setbkcolor(DARKGRAY);

int rightMargin = getmaxx() - 15;
outtextxy(rightMargin - textwidth(timeBuffer),
            getmaxy() - 38, timeBuffer);
outtextxy(rightMargin - textwidth(dateBuffer),
            getmaxy() - 20, dateBuffer);

setbkcolor(oldBk);  // Restore background

Mengapa simpan dan restore bkcolor? setbkcolor() mengubah warna latar teks secara global. Jika tidak di-restore, modul lain yang menggambar teks setelahnya akan terkena dampaknya. Ini adalah defensive programming untuk mencegah bug yang sulit dilacak.

6. Konsep C++ & WinBGIm

  • time_t & struct tm — Tipe data waktu C. time() mengembalikan timestamp UNIX (detik sejak 1970). localtime() mengkonversinya ke struktur dengan field jam, menit, hari, dll.
  • sprintf() — Format string gaya C. %02d = integer 2 digit dengan leading zero. %04d = integer 4 digit.
  • textwidth() — Mengembalikan lebar teks dalam piksel. Berguna untuk alignment (right-align, center).
⚠ Perhatikan Offset Waktu C

tm_mon bernilai 0–11 (Januari = 0), jadi perlu + 1. tm_year adalah tahun sejak 1900, jadi perlu + 1900. Lupa offset ini adalah bug yang sangat umum.

7. Saran Perbaikan

  • sprintf → snprintf — sprintf tidak mengecek batas buffer. snprintf(timeBuffer, sizeof(timeBuffer), ...) mencegah buffer overflow.
  • Pisahkan format dari draw — Format string bisa dipisah ke fungsi getTimeString() agar bisa di-reuse.

Ringkasan clock.cpp

Jam digital di pojok kanan taskbar dengan format HH:MM dan DD/MM/YYYY. Fungsi isTimeChanged() memungkinkan main loop hanya me-redraw jam ketika menit benar-benar berubah (optimasi: 1 redraw per menit, bukan 60 per detik).

mouse.cpp — Input Mouse & Keyboard

1. Tujuan File

Menangani input mouse (klik kiri pada tombol Start, item menu, area luar menu) dan input keyboard (tombol Windows untuk toggle Start Menu).

2. Penjelasan #include

IncludePenjelasan

3. Fungsi handleMouse()

Tujuan

Memproses satu event klik kiri mouse per frame.

Nilai Kembali

true jika ada aksi yang memerlukan redraw (menu dibuka/ditutup, program keluar).

Alur Kerja Langkah Demi Langkah

  1. Cek event — ismouseclick(WM_LBUTTONDOWN) mengecek apakah ada klik kiri dalam antrean. Jika tidak, return false.
  2. Ambil koordinat — getmouseclick(WM_LBUTTONDOWN, x, y) mengambil posisi klik DAN menghapus event dari antrean.
  3. Cek tombol Start — Jika klik di area (10-80, getmaxy()-40 sampai getmaxy()-10), toggle Start Menu.
  4. Cek "Turn Off Computer" — Jika menu terbuka dan klik di area tombol, set isRunning = false.
  5. Cek klik luar menu — Jika menu terbuka dan klik di luar area menu, tutup menu (startMenuOpen = false).
ℹ Kenapa koordinat "Turn Off Computer" di-hardcode?

Koordinat tombol di mouse.cpp (250-400, menuY+410 s/d menuY+440) di-hardcode, terpisah dari startmenu.cpp yang menggambarnya. Ini adalah masalah coupling: jika posisi tombol diubah di startmenu.cpp, mouse.cpp harus di-update manual. Idealnya, koordinat didefinisikan di satu tempat (konstanta) yang di-share.

4. Fungsi handleKeyboard()

Tujuan

Mendeteksi penekanan tombol Windows (VK_LWIN / VK_RWIN) untuk toggle Start Menu.

Nilai Kembali

true jika tombol Windows ditekan (perlu redraw).

Alur Kerja

  1. kbhit() mengecek apakah ada tombol dalam buffer keyboard.
  2. getch() membaca kode tombol.
  3. Jika kode == VK_LWIN atau VK_RWIN, toggle Start Menu.

5. Blok Kode Penting

cpp
if (!ismouseclick(WM_LBUTTONDOWN))
    return false;  // Tidak ada event, skip

int x, y;
getmouseclick(WM_LBUTTONDOWN, x, y);  // Ambil + hapus event

// Cek bounding box tombol Start
if (x >= 10 && x <= 80 &&
    y >= getmaxy() - 40 &&
    y <= getmaxy() - 10)
{
    toggleStartMenu();
    return true;
}

// Klik luar menu = tutup menu
if (startMenuOpen) {
    if (x < 0 || x > 480 || y < menuY || y > startY) {
        startMenuOpen = false;
        return true;
    }
}

6. Saran Perbaikan

  • Gunakan konstanta bernama — const int START_BTN_X = 10; dll. Saat ini koordinat hardcode tersebar di multiple file.
  • Buat struct untuk bounding box — struct Rect { int x, y, w, h; bool contains(int px, int py); }; akan membuat hit test lebih bersih.
  • Hapus #include <stdlib.h> jika tidak digunakan.

utils.cpp — State Global & Utilitas

1. Tujuan File

Menyimpan variabel state global yang di-share antar modul, serta fungsi pembantu untuk deteksi perubahan state. File ini adalah "papan pengumuman" program.

2. Penjelasan #include

IncludePenjelasan
#include "../include/utils.h"Header sendiri (deklarasi extern).
#include <time.h>Untuk fungsi hasTimeChanged() yang menggunakan time() dan localtime().

3. Variabel Global (Definisi)

Ini adalah definisi (bukan hanya deklarasi) dari variabel yang di-extern di utils.h. Tanpa definisi ini, linker akan error (undefined reference).

cpp
bool startMenuOpen = false;  // Start Menu terbuka?
bool needsRedraw   = true;   // Perlu redraw? (awal: ya)
bool isRunning     = true;   // Program masih berjalan?

4. Variabel Global Statis

cpp
static int   lastHour        = -1;
static int   lastMinute      = -1;
static bool  lastMenuState   = false;
static float lastAnimProgress = 0.0f;

Variabel-variabel ini menyimpan state sebelumnya untuk perbandingan di hasTimeChanged() dan hasMenuStateChanged().

5. Fungsi-fungsi

hasTimeChanged()

Sama dengan isTimeChanged() di clock.cpp. Mengecek apakah jam/menit berubah.

⚠ Duplikasi Fungsi

hasTimeChanged() di utils.cpp dan isTimeChanged() di clock.cpp melakukan hal yang sama persis tetapi dengan variabel state yang berbeda. Ini adalah duplikasi kode yang tidak perlu. main.cpp menggunakan isTimeChanged() dari clock.h, sementara hasTimeChanged() di utils.cpp mungkin tidak pernah dipanggil (dead code).

hasMenuStateChanged()

Membandingkan startMenuOpen dengan lastMenuState. Return true jika berubah.

updateLastState()

Menyimpan state saat ini sebagai state terakhir: lastMenuState = startMenuOpen.

6. Konsep C++

  • extern vs definisi — extern di header = "variabel ini ADA di file lain". Definisi di .cpp = "inilah variabelnya". Linker menghubungkan keduanya.
  • Global state — Pola sederhana untuk berbagi data antar modul. Kelemahan: coupling tinggi, sulit di-test, rawan bug.

7. Saran Perbaikan

  • Hapus duplikasi — Pilih salah satu: hasTimeChanged() atau isTimeChanged(), jangan keduanya.
  • Gunakan class atau struct untuk state — struct AppState { bool menuOpen; bool running; }; lalu oper sebagai parameter. Ini mengurangi coupling global.
  • Hapus needsRedraw jika tidak digunakan di kode lain (terlihat hanya di-set tapi tidak pernah di-baca di main.cpp yang punya mekanisme redraw sendiri).

File Header (.h) — Penjelasan Lengkap

Setiap file header menggunakan include guard (#ifndef / #define / #endif) untuk mencegah deklarasi ganda.

Mengapa Include Guard?

Bayangkan skenario: main.cpp meng-include desktop.h dan taskbar.h. Ternyata taskbar.h juga meng-include desktop.h. Tanpa include guard, drawDesktop() akan dideklarasikan dua kali — menyebabkan error kompilasi. Include guard memastikan setiap header hanya diproses sekali.

Saran Perbaikan Keseluruhan

1. Arsitektur

  • Kurangi global state — Variabel global (startMenuOpen, isRunning) menciptakan coupling tersembunyi. Gunakan struct AppState dan oper sebagai parameter ke setiap modul.
  • Pisahkan game loop — Pindahkan logika game loop dari main() ke fungsi run() atau kelas Application.
  • Gunakan namespace — namespace Desktop { void draw(); } akan mengorganisasi kode lebih baik.

2. Kualitas Kode

  • Hapus duplikasi — hasTimeChanged() vs isTimeChanged(), array ikon yang di-hardcode.
  • Gunakan konstanta bernama — const int MENU_WIDTH = 480; akan membuat kode lebih mudah dibaca dan di-maintain.
  • Ganti sprintf dengan snprintf — Mencegah buffer overflow.
  • Ganti malloc dengan new atau std::vector — Lebih idiomatis C++.

3. Animasi

  • Delta time — Gunakan clock() untuk menghitung waktu antar frame. Ini akan membuat animasi konsisten di semua monitor (60Hz, 144Hz, dll).
  • Easing functions — Ganti linear interpolation dengan ease-out untuk animasi yang lebih natural.

4. Input Handling

  • Definisikan bounding box di satu tempat — Koordinat tombol Start dll di-hardcode di mouse.cpp DAN startmenu.cpp. Gunakan konstanta atau struct yang di-share.
  • Event queue yang lebih baik — Saat ini hanya satu klik per frame yang diproses. Jika user mengklik cepat, beberapa klik bisa terlewat.
